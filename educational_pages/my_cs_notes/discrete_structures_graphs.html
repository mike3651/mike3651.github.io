Graphs

What is a graph? A graph is a set of vertices all linked together by a set of edges. Graphs are not very difficult to find in the real world. We find them when we network with people to see who we know, routing between cities to see how long it takes to get from one city to another, tournament brackets to see what teams play what and several other areas in life.

Common graphs

  • Simple: Each edge connects two different vertices and no two edges connect the same pair of vertices.
  • Multigraph: A graph in which there can be multiple edges connecting the same vertices.
  • Pseudograph: Graph that can include loop and multiple edges to other vertices or to a single vertex.
  • Undirected: Graph in which the each edge doesn't care as to what the start or endpoint vertices.
  • Directed: Each edge is associated with an ordered pair of vertices. Such that Edge(u,v) \(\neq\) Edge(v,u).
  • Simple directed: Directed graph with no loops nor multiple edges.
  • Directed multigraph: Directed graph that contains multiple directed edges.
  • Mixed: Graph which contains directed and undirected edges.
  • Bipartite: Graph in which its edges can be separated into two sets such that \(V_1, V_2 \subset V\) and no two vertices \(v_i\in V_1, v_i\in V_2 where i >= 1\) are connected to one another.
  • Complete: Simple graph that where there is only one edge connecting each pair of distinct vertices

Graphing algorithms you should know

Depth First Search


Goes through all of the nodes in a graph
input: The node to start at
output: None

DFS(startingNode)
   create a stack of type node
   add starting node to the stack
   while stack is not empty
      pop node from the stack and mark as visited
      for each neighbor of popped node
         if neighbor is not visited
            add neighbor to the stack

Breadth First Search


Goes through all of the nodes in a graph
input: The node to start at
output: None

BFS(startingNode)
   create a queue of type node
   add starting node to the queue
   while queue is not empty
      remove node from the stack and mark as visited
      for each neighbor of remove node
         if neighbor is not visited
            add neighbor to the queue

Bipartite coloring


Colors a graph in such a way that the colors represent a subset
input: graph G where g is any sort of graph
output: colored graph if it is possible to make a bipartite

makeBipartite(G)

         if v is not colored
            v->color = color
         else if v->color is not equal to color
            return

Useful definitions & theorems

Neighbors of a subset of vertices

Assuming that A is a subset of V and v is a vertex in V, we can represent the neighbors in the subset A is:
\(N(A) = \cup_{v \in A}N(v)\)

Easy check for nonIsomorphism

If the maximum degree in either graph H or G are different then the two graphs are not isomorphic. In order for the graphs to be isomorphic there must exists a one-to-one function such that if vertex a & b are adjacent in G then f(a) & f(b) are adjacent in H

Terminology

  • Loop: A vertex which has an edge to itself.
  • Neighbor: A vertex which is connected to another unique vertex.
  • Isomorphism: You can translate one graph into another one.
